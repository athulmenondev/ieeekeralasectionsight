# Runtime Rules

1. Never hardcode content.

2. All content must originate from /data.

3. Treat /data as a lightweight CMS.

4. Every page must render dynamically.

5. Use React + Vite.

6. Use TypeScript.

7. Use Tailwind CSS.

8. Use SCSS for reusable design primitives.

9. Shared components only.

10. Footer appears globally.

11. Footer must include:
   - IEEE Privacy Policy
   - IEEE Terms and Conditions

12. All parallax elements must use:

will-change: transform;

13. Accessibility:
WCAG 2.1 AA

14. Mobile First
